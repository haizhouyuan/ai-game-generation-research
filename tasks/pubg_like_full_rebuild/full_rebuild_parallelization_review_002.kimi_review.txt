### 2. Missing Acceptance Gates (Where a Weak Upgrade Can Slip Through)

These are gaps in the current gate list that would allow a "we generated some stuff" outcome to be mistaken for a full rebuild:

**A. Reference-to-Asset Chain Completeness Gate**
- **Gap:** The gates count "generated asset packets" but do not mandate that each packet's `source/reference.md` must contain a SHA256 linking it to a specific reference image. Without this, someone could import a free Sketchfab model, run it through Blender, and call it "generated."
- **Fix:** Every packet in `model/optimized.glb` must have a `source/reference.md` with a verifiable SHA256 of the reference image that fed the generator. Gate fails if `reference_sha256` is missing or mismatched.

**B. Hero Asset No-Procedural-Fallback Gate**
- **Gap:** The runtime could silently fall back to old procedural boxes if a GLB fails to load. The current gates mention "fail-closed fallback" but don't define the enforcement mechanism.
- **Fix:** W8 gate must assert `material_map_count >= 3` (basecolor, normal, roughness) for hero rifle, player character, and enemy character. If the count is zero, the build fails — no silent fallback to `MeshStandardMaterial` with flat color.

**C. Animation Integrity Gate**
- **Gap:** W5 mentions "retarget animation set" but there is no gate checking that the new character mesh actually deforms correctly with idle/walk/aim/reload animations. A beautiful static GLB that breaks on skinning is worthless for a playable slice.
- **Fix:** After character replacement, run a Blender script that plays the animation clip and exports frames. Gate passes only if vertex displacement is non-zero and bone weights are present on >95% of vertices.

**D. Polycount / LOD Gate (Partial overlap with Gemini, but stricter)**
- **Gap:** No hard triangle budgets.
- **Fix:** Define and enforce:
  - Props/loot/clutter: < 5k tris
  - Hero rifle: < 15k tris
  - Characters: < 50k tris
  - Environment ground plane: < 2k tris
  Every packet must include `reports/polycount.json`. Gate fails if exceeded — no exceptions.

**E. PBR Consistency / Rainy HDRI Test Scene Gate**
- **Gap:** Individual assets may look good in isolation but mismatched under unified lighting.
- **Fix:** Create a single `lighting_test_scene.html` with the rainy checkpoint environment. All final assets are dropped into this scene and captured from the same camera angles. Visual comparison (eyeball + SSIM) ensures albedo ranges and roughness curves are cohesive.

**F. Decal/Clutter Density Gate**
- **Gap:** W6 lists decals and clutter but the completion criteria don't specify *how many* or *what coverage*.
- **Fix:** Minimum 5 distinct decal types (mud, scratch, impact, warning marking, oil) and 5 clutter props (casing, paper, cable, cone, pallet) must be present in the final micro-scene screenshot. Gate fails if the scene looks "clean."
