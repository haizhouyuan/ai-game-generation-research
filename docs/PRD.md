# PRD: AI-Assisted 3D Game Generation Research

## Goal

Determine which current AI-assisted toolchains can help produce a small 3D game prototype from concept to playable loop, and identify the most promising path to try first.

## Target User

A parent/operator helping a child explore game creation with safe, controllable, inspectable tooling.

## Research Questions

1. Which parts of a 3D game can be generated reliably today?
2. Which parts still require manual modeling, engine work, or scripting?
3. Which tools can run locally on HomePC without uncontrolled proxy downloads?
4. Which tools are practical for a child-friendly creative workflow after adult setup?
5. Which path gives the best chance of a small playable prototype without locking into one proprietary hosted stack?

## Output Standard

Each promoted path must include source evidence, setup notes, commands, generated assets or project files, failure notes, and a decision.

## Machine Placement

- YogaS2: controller, docs, light research, short probes.
- HomePC: GPU/CUDA/model-heavy generation trials.
- M9: CLI/build/read-only probes now; GUI/browser/editor validation only after GUI health is repaired or bypassed.
