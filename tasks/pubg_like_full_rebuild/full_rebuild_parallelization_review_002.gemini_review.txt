### 2. Missing Acceptance Gates (Risks for a Weak Upgrade)
The current gates ensure assets are generated and textured, but miss critical runtime and consistency checks:
*   **Polycount / Performance Gate:** There is no maximum vertex/triangle count specified. Hunyuan/TRELLIS outputs can be massively dense. Without a hard decimation/polycount gate (e.g., `< 10k tris for props, < 50k for hero`), the Three.js scene will crash or stutter, failing the playable slice requirement.
*   **Collision/Physics Proxy Gate:** Replacing procedural boxes with complex 3D meshes requires updated physics. There is no gate ensuring the generated visual asset has a matching, simplified collision mesh for the tactical game engine.
*   **PBR Consistency/Lighting Gate:** Assets generated from different models (Hunyuan vs. TRELLIS vs. ComfyUI) will have mismatched albedo/roughness curves. You need an environment integration gate (e.g., a standardized rainy HDRI test scene) to prove the materials look cohesive together, not just individually.
