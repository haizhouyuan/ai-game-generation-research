## 1. Top-Level Registry Fields

```json
{
  "schema_version": "3.0.0",
  "generated_at": "<ISO8601>",
  "experiment": "<string, experiment name>",
  "source": "<string, registry origin description>",
  "registry_id": "<string, UUID>",
  "parent_registry": "<string, path to prior registry if applicable>",
  "assets": [ ... ],
  "_meta": {
    "hero_asset_count": "<integer>",
    "total_asset_count": "<integer>",
    "pbr_complete_count": "<integer>",
    "blocked_count": "<integer>"
  }
}
```

**Validation notes:**
- `schema_version` must be `3.0.0` exactly
- `generated_at` must be valid ISO8601 with timezone
- `experiment` must match the directory name pattern `experiments/<name>/`
- `assets` must be non-empty array

## 2. Per-Asset Required Fields

### Base fields (all asset classes)

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| `asset_id` | string | yes | lowercase_snake, no spaces, unique within registry |
| `asset_class` | enum | yes | one of: `weapon`, `character`, `material`, `vehicle`, `prop`, `environment` |
| `role` | string | yes | human-readable purpose, â‰Ą10 chars |
| `status` | enum | yes | one of: `baseline_only`, `target`, `in_progress`, `production_ready`, `rejected` |
| `paths` | object | yes | see Per-Asset Paths section |
| `provenance` | object | yes | see Provenance section |
| `geometry` | object | yes | see Geometry section |
| `materials` | object | yes | see Materials section |
| `quality_gates` | object | yes | see Quality Gates section |
| `evidence` | object | yes | see Evidence section |
| `hashes` | object | yes | see Hashes section |
| `blockers` | array | yes | must be empty array for `production_ready` status |

**weapon**

## 3. Per-Asset Paths

```json
"paths": {
  "model": "<string, absolute path, .glb extension required>",
  "preview": "<string, absolute path or null>",
  "textures": {
    "basecolor": "<string, absolute path or null>",
    "normal": "<string, absolute path or null>",
    "roughness": "<string, absolute path or null>",
    "metallic": "<string, absolute path or null>",
    "ao": "<string, absolute path or null>",
    "emissive": "<string, absolute path or null>",
    "opacity": "<string, absolute path or null>"
  },
  "collision_proxy": "<string, absolute path or null>",
  "lod_levels": {
    "lod0": "<string, path>",
    "lod1": "<string, path or null>",
    "lod2": "<string, path or null>"
  }
}
```

**Validation notes:**
- `model` extension must be `.glb` for `production_ready`
- texture paths must have valid image extensions: `.png`, `.jpg`, `.jpeg`, `.ktx2`
- `collision_proxy` must be `.glb` or `.gltf` if present
- All paths must be absolute and within the experiment directory

## 4. Provenance

```json
"provenance": {
  "source": "<string, source description, â‰Ą5 chars>",
  "license_risk": "<enum: local_only | permissive | restricted | unknown>",
  "generation_route": "<string, route name, e.g. Route A StableGen/Blender-first>",
  "reference_images": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>",
      "description": "<string>"
    }
  ],
  "generation_prompts": [
    {
      "model": "<string, model name>",
      "prompt": "<string>",
      "params": "<object, model-specific params>",
      "seed": "<integer or null>",
      "output_path": "<string>"
    }
  ],
  "blender_version": "<string, e.g. 4.2.0>",
  "cleanup_report": "<string, absolute path or null>",
  "notes": "<string, free text>"
}
```

**Validation notes:**
- `license_risk` of `unknown` blocks `production_ready` for hero assets
- `reference_images[].sha256` must be valid 64-char hex
- `cleanup_report` must be present for all `production_ready` assets
- `generation_prompts` must not be empty for generated assets

## 8. Evidence

```json
"evidence": {
  "gameplay_screenshots": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>",
      "description": "<string>",
      "timestamp": "<ISO8601>"
    }
  ],
  "preview_renders": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>",
      "resolution": "<string, e.g. 1920x1080>"
    }
  ],
  "material_reports": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>"
    }
  ],
  "blender_screenshots": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>",
      "view": "<string, e.g. material_preview, wireframe>"
    }
  ],
  "threejs_runtime_screenshots": [
    {
      "path": "<string, absolute path>",
      "sha256": "<string, 64-char hex>",
      "scene_setup": "<string>"
    }
  ],
  "animation_clips_proof": {
    "path": "<string, absolute path or null>",
    "clips_proven": ["<string>"]
  }
}
```

**Validation notes:**
- `gameplay_screenshots` must have at least 1 entry for `production_ready`
- `blender_screenshots` must include at least `material_preview` view for `production_ready`
- `threejs_runtime_screenshots` required for hero assets
- `animation_clips_proof.path` must be non-null for characters with `production_ready` status
- All sha256 values must be valid 64-char hex

## 11. Hero Asset Fail-Closed Behavior

For assets with `role` containing `hero` or `production` (case-insensitive):

| Requirement | Enforcement |
|-------------|--------------|
| `provenance.license_risk` must not be `unknown` | schema validation |
| `provenance.cleanup_report` must be present | schema validation |
| `quality_gates.fallback_allowed_in_evidence` must be `false` | schema validation |
| `materials.pbr_complete` must be `true` | schema validation |
| `materials.pbr_consistency.*` must all be `true` | schema validation |
| `fail_closed.hero_asset` must be `true` | schema validation |
| `fail_closed.enforced` must be `true` | schema validation |
| `blockers` must be empty array | schema validation |
| `evidence.threejs_runtime_screenshots` must be non-empty | schema validation |

## 12. Cross-Asset Validation Checks

1. **Duplicate asset_id**: no two entries may share the same `asset_id`
2. **Orphaned paths**: every path in `paths.*` must resolve to an existing file
3. **Reference image integrity**: every `provenance.reference_images[].sha256` must match the file on disk
4. **Route consistency**: `provenance.generation_route` must match a known route in the experiment's route registry
5. **Animation clip alignment**: `animations.clips` entries must have corresponding proof in `evidence.animation_clips_proof.clips_proven`
6. **Parent registry inheritance**: if `parent_registry` is set, all `baseline_only` assets must exist in the parent
7. **Texture map coverage**: for `materials.pbr_complete == true`, all five base texture paths (`basecolor`, `normal`, `roughness`, `metallic`, `ao`) must be non-null
